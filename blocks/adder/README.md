# Adder Benchmark Blocks (A64: 64-bit, LSDL + matched CMOS)

Structural netlist generators for the adder benchmark pair. The taped-out
width is **64 bits** (decision A64, 2026-06-03; 4/16-bit variants kept as
regression vehicles). Both flavors implement the **same systolic ripple
dataflow** — carry advances one bit per cycle, outputs time-skewed
(`sum[i]` valid at cycle i+1) — so the comparison isolates the logic
family, not the adder architecture. Generated — do not hand-edit the
`.v` files.

## Why generators, not Yosys

Every LSDL cell is a positive-edge flop (precharge CLK=0, evaluate CLK=1).
Yosys/ABC technology mapping targets *combinational* cells; it cannot map
onto a sequential-only library. The CMOS twin is generated too so the
dataflow graphs match exactly.

## The two flavors

| | LSDL (`gen_lsdl_adder.py`) | CMOS (`gen_cmos_adder.py`) |
|---|---|---|
| Library | `lsdl_fd_sc_9t5v0` | stock `gf180mcu_fd_sc_mcu9t5v0` |
| Rails | dual (every cell inverts) | single |
| Clocking | C1/C2 two-phase, 1 stage per half period | one clock, posedge |
| Slice | 7 aoi22/inv per bit (XOR + dual carry via majority self-duality) | 1 `addf_1` + DFFs |
| Cells @ 64 b | 16,832 (16,512 inv + 320 aoi22) | 4,224 (4,160 dffq_1 + 64 addf_1) |
| Cell area @ 64 b | 1.40 mm² | 0.334 mm² |
| Latency | sum[i] @ 2i+2 half-cycles = i+1 cycles | sum[i] @ i+1 cycles (identical) |
| Timing (pre-PnR, TT) | **1 GHz closes, +0.134 ns slack** | **f_max ≈ 634 MHz** (+0.42 ns @ 2.0 ns) |

### LSDL architecture — dual-rail pipelined ripple-carry

Inverting-only cells flip a single-rail signal's polarity every stage, so:

- **Operands enter dual-rail** (`a`/`an`, `b`/`bn`, `cin`/`cinn`) and are
  staged through paired `lsdl_inv_x1` chains; true/false roles swap each
  stage (generator tracks parity).
- **XOR in one cell:** `x = aoi22(a,b,an,bn) = !(ab + na·nb)`.
- **Carry rails** exploit majority self-duality (`maj(a,b,c) = !maj(!a,!b,!c)`
  and `!a^!b = a^b = x`):
  - `nc[i+1] = aoi22(a, b, c, x)`
  - `c[i+1]  = aoi22(an, bn, nc, x)`
- **Sum:** `s = aoi22(x, c, nx, nc) = x^c`.

2 stages per bit; cell count `4W² + 7W`, dominated by operand staging
inverters. Full schematic + signal glossary + C1/C2 timing:
`adder_slice_schematic.pdf` (generated by `draw_adder_slice.py`).

### CMOS architecture — same graph, native idiom

One `addf_1` full adder per bit between DFF pipeline ranks; operand bit i
delayed i cycles by a `dffq_1` chain; carry and sum registered per cycle.
Single-rail (CMOS inverts freely) — the dual-rail tax is an LSDL-only cost
and the comparison should show it.

## Files

|------|------|

## Verification status (2026-06-03)

- **Selftests:** 200 random vectors PASS — LSDL @ W=4/16/64,
  CMOS @ W=16/64 (cycle-accurate flop-per-cell simulation).
- **OpenSTA 3.1.0** (measured-PVT Liberty for LSDL; stock TT lib for CMOS):
  - LSDL-64 @ 1 GHz: worst setup **+0.134 ns** (aoi22 266 ps + setup
    100 ps vs 500 ps half-period), hold +0.5 ns, TNS 0.
  - CMOS-64 @ 2.0 ns: worst setup +0.42 ns, hold +0.09 ns, TNS 0
    → f_max ≈ 634 MHz (ideal clock, pre-PnR).

```bash
python3 gen_lsdl_adder.py --width 64 --selftest --out adder64_lsdl.v
python3 gen_cmos_adder.py --width 64 --selftest --out adder64_cmos.v
./lsdl_lib/scripts/run_in_container.sh "ADDER_W=64 sta -no_init -exit $PWD/run_sta_adder.tcl"
./lsdl_lib/scripts/run_in_container.sh "CMOS_PERIOD=2.0 ADDER_W=64 sta -no_init -exit $PWD/run_sta_cmos.tcl"
```
