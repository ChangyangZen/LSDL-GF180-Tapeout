# Adder Tester Pair (TST)

Identical go/no-go testers for the 64-bit adder pair. One RTL module,
two instances:

| | LSDL flavor | CMOS flavor |
|---|---|---|
| Pins | c1, c2, result[64:0], correct, incorrect | c1, result[64:0], correct, incorrect |
| DUT | `lsdl_adder_wrap` (adder + dual-rail invs) | `cmos_adder64` |

The two testers are the same netlist; the LSDL instance's `c2` only routes
to the adder. Static-CMOS (stock 9T cells), always-on domain.

## Operation
1. Drive vector k of 10 (`vectors.vh`, generated by `gen_vectors.py`:
   carry-ripple worst cases + zero + 5 PRNG vectors).
2. FSM ticks once per 6 c1-cycles (3-stage Johnson counter, self-starting,
   no reset pin). Vector held 16 ticks = 96 cycles.
3. Result sampled at tick 11 = cycle 66 — the exact first arrival in BOTH
   flavors (apply 1 + W+1). At-speed corruption is caught, not masked.
4. After 10 vectors: refresh LEDs — correct = all matched, else incorrect.
   Loops forever (~1 kHz at 1 GHz clock — LEDs steady).

## f_max measurement
Sweep external clock until `incorrect` lights: that's the adder's silicon
f_max. Tester margin per design: full-rate logic is flop-limited
(divider +0.02 ns @ 0.9 ns TT); FSM is multicycle ÷6 with all fanouts
≤32. Tester ceiling ~1.1 GHz (TT).

## Files
- `adder_tester.v` — tester core (DIV=6, HOLD=16, exact-arrival sample)
- `gen_vectors.py` → `vectors.vh` — vector ROM
- `lsdl_adder_wrap.v` — adder + 129 boundary inverters (keeps testers identical)
- `tb_tester_cmos.v` — tester + REAL cmos_adder64 netlist (stock models)
- `tb_tester_lsdl.v` — tester + behavioral LSDL latency model
- `synth_tester.ys`, `run_sta_tester.tcl` — synthesis + timing

## Verification (2026-06-03)
- iverilog: CMOS netlist PASS · negative control (+corrupt) PASS · LSDL model PASS
- Yosys: 1,055 stock cells, ~0.04 mm²
- OpenSTA at 0.9 ns: divider +0.02; FSM paths violated pre-PnR
  (Yosys/ABC does not buffer or upsize).
- **PnR (OpenROAD, `pnr/run_pnr_tester.tcl`): CLOSED at 0.9 ns —
  worst setup +0.012 ns, hold +0.49 ns, TNS 0, route DRC 0.**
  `repair_design` buffer trees fixed what synthesis could not, as
  predicted. Tester ceiling ≈ 1.11 GHz (TT) — above the LSDL adder's
  1 GHz operating point and 2.2× the CMOS adder's 500 MHz.

## Placement requirement (T14)
Each tester must sit at the SAME distance from its adder (mirrored
placement) so the comparison is fair.
